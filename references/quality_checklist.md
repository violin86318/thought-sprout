# 质量控制清单

## 整体连贯性

- [ ] 开场白是否准确概括了笔记集合
- [ ] 各条笔记之间是否有隐含的关联
- [ ] 整体语言风格是否统一

## 故事质量

- [ ] 每个故事是否有具体的人名、时间、事件
- [ ] 故事是否有戏剧性和启发性
- [ ] 故事与用户笔记的关联是否清晰

## 金句质量

- [ ] 每个Aha瞬间是否有反直觉的洞察
- [ ] 金句是否有诗意和哲理性
- [ ] 金句是否与故事内容紧密相关

## 引用准确性

- [ ] 所有人名、书名、理论名是否准确
- [ ] 历史事件的时间、数据是否可靠
- [ ] 引用来源是否权威

## 语言风格

- [ ] 是否避免了AI腔调
- [ ] 是否有文学性和温度
- [ ] 是否保持了适度的克制（不过度煽情）

## 故事质量检查

### 具体性检查
- [ ] 有具体的人名？
- [ ] 有具体的时间（年份）？
- [ ] 有具体的事件或数据？

### 戏剧性检查
- [ ] 有冲突或转折？
- [ ] 有意外的结果？
- [ ] 有对比或反差？

### 相关性检查
- [ ] 与用户笔记主题强相关？
- [ ] 能够支撑"种子"中的观点？
- [ ] 不是生硬的知识堆砌？

### 准确性检查
- [ ] 历史事件是否准确？
- [ ] 人名、书名是否正确？
- [ ] 数据来源是否可靠？

## 金句质量检查

### 结构检查
- [ ] 有破折号、冒号或逗号制造停顿？
- [ ] 使用了对比、悖论或本质揭示的结构？

### 洞察检查
- [ ] 有反直觉的观点？
- [ ] 不是陈词滥调？
- [ ] 与故事内容紧密相关？

### 语言检查
- [ ] 有诗意或哲理性？
- [ ] 不过度煽情？
- [ ] 长度适中（20-40字）？

## 整体风格检查

### 文学性
- [ ] 语言有美感，但不矫情
- [ ] 比喻新鲜，但不晦涩
- [ ] 有温度，但不煽情

### 连贯性
- [ ] 开场白准确概括笔记集合
- [ ] 各条扩展之间有隐含关联
- [ ] 整体语言风格统一

### 克制性
- [ ] 不过度解读用户笔记
- [ ] 不说教或居高临下
- [ ] 不使用夸张的表达
